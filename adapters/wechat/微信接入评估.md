# 微信接入评估报告（V2.4）

## 评估结论：暂缓接入，继续使用飞书 + Telegram 作为主入口

### 一、三个微信方案对比

| 方案 | 难度 | 个人可用性 | 需要公网 | 成本 | 限制 |
|---|---|---|---|---|---|
| 微信公众号 | 中 | ✅ 可以 | ✅ 需要 | 免费 | 被动回复，用户必须主动发消息；需要认证服务号才能主动发消息模板 |
| 企业微信（自建应用） | 中高 | ⚠️ 个人可以用但有点绕 | ✅ 需要 | 免费（50人以下） | 需要注册企业，个人也能注册但流程稍麻烦；API 能力强 |
| 微信个人号（WeChaty/itchat） | 高 | ⚠️ 有封号风险 | ❌ 不需要 | 免费 | 非官方，违反微信 TOS，容易封号 |
| **推荐：飞书 + Telegram** | 低 | ✅ 很好用 | Telegram ❌ 不需要；飞书 ✅ 需要 | 免费 | 官方 API，稳定，个人用完全没问题 |

### 二、各方案详细评估

#### 1. 微信公众号（订阅号/服务号）
- **优点**：用户基数大，你肯定每天都开微信
- **缺点**：
  - 订阅号没有主动发消息权限，只能用户先发你 48 小时内回复
  - 服务号需要企业/个体户认证（个人可以注册但认证要钱）
  - 消息回调需要公网 HTTPS 地址
  - 消息类型有限制，交互卡片不如飞书灵活
- **个人使用推荐度**：⭐⭐

#### 2. 企业微信（自建应用）
- **优点**：API 能力强，支持主动发消息，支持消息卡片
- **缺点**：
  - 需要注册企业（个人可以注册，不需要真的有公司）
  - 个人微信接收企业微信消息需要在企业微信 App 里，或者关联微信插件
  - 同样需要公网 HTTPS 回调
  - 配置比较复杂
- **个人使用推荐度**：⭐⭐⭐

#### 3. 微信个人号（非官方库）
- **优点**：直接用自己微信，不需要额外 App
- **缺点**：
  - ❗️ 封号风险极高，WeChaty 现在也需要付费 token 了
  - 非官方 API，随时可能失效
  - 违反微信用户协议，个人号被封得不偿失
- **个人使用推荐度**：❌ 不推荐

### 三、当前方案对比（飞书 + Telegram）

| 场景 | 飞书 | Telegram |
|---|---|---|
| 国内网络 | ✅ 原生支持 | ⚠️ 需要代理 |
| 不需要公网 | ❌ 需要回调地址 | ✅ 长轮询即可 |
| 交互卡片/按钮 | ✅ 非常完善 | ✅ Inline Keyboard 也支持 |
| 消息提醒 | ✅ 手机通知很及时 | ✅ 手机通知很及时 |
| 配置难度 | 低 | 极低 |

### 四、建议

**当前阶段（V2.4）结论：暂缓微信接入**，理由：
1. 飞书 + Telegram 已经覆盖了「公司网络（飞书）」和「出门在外/海外（Telegram）」两个主要场景
2. 微信三个方案都有各自的明显缺点，投入产出比不高
3. 个人使用场景下，飞书+Telegram 完全够用，不需要为了「在微信里用」增加复杂度

**如果未来真的需要微信：优先选企业微信自建应用**，因为 API 能力最完整、最稳定，个人也能注册企业，不需要营业执照。

### 五、什么时候可以再考虑微信？
- 飞书/Telegram 有明显使用痛点
- 需要给家人/朋友用（他们没有飞书/Telegram）
- 需要接入微信生态的内容发布
- 有稳定的公网 HTTPS 环境了
