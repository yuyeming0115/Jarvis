## 图片生成 HTTP 405 修复（17:20）
- **问题**：AI文生图报 HTTP 405，因为 `generate_image()` 复用 TinyRouter base_url 调 `/images/generations`，但 TinyRouter 不支持此路由
- **根因分析**：TinyRouter 是 LLM 文本网关，仅代理 `/chat/completions`，不支持 OpenAI `/images/generations` 图片接口
- **修复方案**：图片生成改走独立配置的 `image_gen_base_url`，不再复用 TinyRouter：
  - `llm.py`：新增 `_get_image_gen_base_url()` / `_get_image_gen_api_key()` 读独立设置
  - `generate_image()` 改用图片专用 URL + Key
  - `is_image_gen_configured()` 改为检查 `image_gen_base_url`
  - `main.py`：`/api/llm/status` 的 `image_configured` 独立检测
  - `app.js`：前端提示文案通用化（去掉 AGNES_API_KEY 硬编码）

## 图片生成内容安全容错机制（17:42）

### 用户反馈
生图能跑了，但长提示词/复杂中文提示词会触发 Agnes 的 `content_policy_violation` (HTTP 400)，导致自动化流水线卡住。

### 实现的容错方案
**`llm.py` 改动**：
1. 新增 `_clean_image_prompt()` — 正则清洗敏感词（数字人、政治、暴力、色情、8K参数等）
2. 新增 `_simplify_prompt()` — 激进简化：截断到 200 字符以内
3. `generate_image()` 新增自动重试循环（默认 2 次）：
   - 第 1 次失败 → 清洗敏感词后重试
   - 第 2 次失败 → 截断简化 + 去掉负面提示词后重试
   - 返回结果中附加 `prompt_meta`（记录是否清洗/简化/重试）
4. 新增 `max_retries` 参数控制最大重试次数

**`app.js` 改动**：
- 生图成功后在图片上方显示黄色提示条：提示用户哪些内容被移除/简化了、重试了几次
