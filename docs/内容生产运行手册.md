# Jarvis 内容生产运行手册 V4.0

> 本文档描述如何使用 Jarvis 内容生产功能，包括草稿生成、审核、归档的完整操作流程。

---

## 1. 功能概述

Jarvis V4.0 提供了一条完整的内容生产流水线：

```
选题（topic）
  ↓
生成草稿（draft）
  ↓
编辑/重写
  ↓
提交审核
  ↓
批准/驳回
  ↓
归档到 Wiki
```

---

## 2. 前置条件

### 2.1 服务启动

```bash
cd /path/to/Jarvis
python -m backend.main
```

服务默认监听 `http://127.0.0.1:8080`。

### 2.2 LLM 配置

打开 Workbench → 右上角 ⚙️ 设置 → **LLM 配置**：
- TinyRouter Base URL：`http://localhost:20129/v1`
- 文本模型组：`text-smart-fallback`
- 图片模型组：`image-smart-fallback`

### 2.3 图片生成配置（可选）

如果需要 AI 文生图功能，在设置面板底部 **图片生成配置** 填写：
- 图片生成 API 端点：`https://api.agnes-ai.com/v1`
- 图片生成 API 密钥：`sk-xxx`
- 图片生成模型：`Agnes Image 2.1 Flash`

## 3. 操作流程

### 3.1 从选题生成草稿

**网页端操作**：
1. 打开 Workbench → **选题** tab
2. 选择一个选题，点击「生成内容」
3. 选择渠道和类型：
   - 渠道：`wechat`（公众号）/ `xiaohongshu`（小红书）/ `video`（视频）
   - 类型：`article`（文章）/ `post`（短文）/ `script`（口播稿）
4. 等待生成完成

**API 调用**：
```bash
curl -X POST http://127.0.0.1:8080/api/topics/{topic_id}/generate-draft \
  -H "Content-Type: application/json" \
  -d '{"channel": "wechat", "content_type": "article", "generation_mode": "auto"}'
```

### 3.2 查看和编辑草稿

**网页端操作**：
1. 打开 **内容草稿** tab
2. 点击草稿标题查看详情
3. 点击「编辑」修改标题、正文、提示词

**API 调用**：
```bash
# 读取草稿
curl http://127.0.0.1:8080/api/drafts/{draft_id}

# 编辑草稿
curl -X PATCH http://127.0.0.1:8080/api/drafts/{draft_id} \
  -H "Content-Type: application/json" \
  -d '{"title": "新标题", "body": "新正文"}'
```

### 3.3 提交审核

草稿生成后，状态为 `draft`，需要提交审核：

**网页端操作**：
1. 打开草稿详情
2. 点击「提交审核」
3. 填写提交说明（可选）

**API 调用**：
```bash
curl -X POST http://127.0.0.1:8080/api/drafts/{draft_id}/submit-review \
  -H "Content-Type: application/json" \
  -d '{"submit_notes": "请审核"}'
```

### 3.4 批准或驳回

**网页端操作**：
1. 打开 **审核** 状态的草稿
2. 点击「批准」或「驳回」
3. 填写审核意见

**API 调用**：
```bash
# 批准
curl -X POST http://127.0.0.1:8080/api/drafts/{draft_id}/approve \
  -H "Content-Type: application/json" \
  -d '{"review_notes": "批准发布"}'

# 驳回
curl -X POST http://127.0.0.1:8080/api/drafts/{draft_id}/reject \
  -H "Content-Type: application/json" \
  -d '{"rejection_reason": "需要补充案例"}'
```

### 3.5 归档到 Wiki

只有 `approved`（定稿）状态的草稿可以归档。

**网页端操作**：
1. 打开已批准的草稿
2. 点击「归档到 Wiki」
3. 系统自动生成 Markdown 文件到 `wiki/pages/content/`

**API 调用**：
```bash
curl -X POST http://127.0.0.1:8080/api/drafts/{draft_id}/archive-wiki \
  -H "Content-Type: application/json" \
  -d '{}'
```

归档文件路径：
```
wiki/pages/content/YYYY-MM-DD-{draft_id}.md
```

## 5. 故障排查

### 5.1 LLM 未配置

**现象**：生成草稿时报错 `LLM 未配置`

**解决**：在设置面板填写 TinyRouter Base URL 和模型组名称

### 5.2 图片生成失败（HTTP 405）

**现象**：AI 文生图返回 `HTTP 405 Method Not Allowed`

**原因**：图片生成 API 地址配置错误

**解决**：在设置面板 **图片生成配置** 区域，确认地址为 `https://api.agnes-ai.com/v1`

### 5.3 内容审核拦截（HTTP 400）

**现象**：图片生成返回 `content_policy_violation`

**原因**：提示词触发了 Agnes AI 内容安全策略

**解决**：
- 系统会自动清洗敏感词并重试
- 如果仍失败，简化提示词（去掉 `8K`、`超高清` 等技术参数）

### 5.4 草稿状态异常

**现象**：无法提交审核/批准/归档

**原因**：草稿状态不符合操作前提

**状态转换规则**：
```
draft → reviewing（提交审核）
reviewing → approved（批准）/ rejected（驳回）
approved → archived（归档）
```

## 6. 目录结构

内容生产功能涉及的文件：

```
Jarvis/
├── backend/
│   ├── main.py                    # API 路由
│   └── core/
│       ├── drafts.py              # 草稿 CRUD
│       ├── content_generator.py   # 内容生成器
│       ├── content_reviews.py    # 审核状态机
│       ├── wiki_archive.py       # Wiki 归档
│       ├── prompts.py            # Prompt 加载器
│       └── llm.py               # LLM 调用（含图片生成）
├── prompts/                      # Prompt 模板
│   ├── draft-wechat-article.md
│   ├── draft-xiaohongshu-post.md
│   ├── draft-video-script.md
│   ├── rewrite-draft.md
│   └── review-rubric.md
├── services/                     # 测试脚本
│   ├── content-smoke-test.sh
│   ├── content-regression-test.sh
│   └── export-drafts.sh
└── wiki/pages/content/          # 归档文件存储
```
