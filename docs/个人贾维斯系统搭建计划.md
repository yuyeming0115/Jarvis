## 二、硬件部署方案

| 设备 | 角色 | 部署内容 | 说明 |
|------|------|----------|------|
| **Mac Mini M4（家里）** | 主力服务器 | OpenClaw + Hermes + TinyRouter + Codex CLI + 全部Skill | 7×24小时开机，永不休眠，功耗5-10W |
| **绿联 DH4300 Plus** | 存储+备份 | 文件存储 + NAS版OpenClaw备机 | 应用中心一键安装OpenClaw，素材/成品/知识库全存NAS |
| **公司 Win 电脑** | 客户端 | 飞书网页版（个人账号）+ 浏览器访问仪表盘 | 不跑服务，飞书网页版跟贾维斯对话，Tailscale访问仪表盘 |
| **手机** | 移动端入口 | 飞书App（切换个人账号）+ Tailscale | 随时随地发灵感、收推送、审稿件 |
| **云服务器** | 可选（后期） | 仅跑公众号发布 | 初期不需要，等流程跑通后再考虑（约24-50元/月） |

### 飞书账号体系说明（重要）

**使用个人飞书账号作为贾维斯专属入口**，不使用公司企业飞书：

| 对比 | 个人飞书（贾维斯用这个） | 公司企业飞书（工作用） |
|------|--------------------------|------------------------|
| 机器人创建 | 开放平台自由创建，无需审批 | 需要IT管理员审批 |
| 数据隐私 | 完全在你个人控制下 | 公司可能有审计权限 |
| 多维表格/云文档 | 个人空间，灵感库/文章/看板都在这里 | 公司工作表格，不直接打通 |
| 使用场景 | 灵感管理、内容生产、工作提醒、个人知识库 | 日常工作沟通、公司任务 |

**双账号共存方式：**

| 设备 | 公司飞书（工作） | 个人飞书（贾维斯） |
|------|------------------|---------------------|
| 公司Win电脑 | 飞书客户端登录企业账号 | 浏览器飞书网页版登录个人账号 |
| 家里Mac | 不需要 | 飞书客户端登录个人账号 |
| 手机 | 飞书App切换到企业账号 | 飞书App切换到个人账号（飞书支持多账号切换） |

**工作任务录入方式（三种，选最方便的）：**
1. **快捷录入（推荐）**：在公司飞书网页版给贾维斯发一句话，如"周五交Q3方案，提醒我"
2. **手动转发**：公司飞书里的重要消息/任务，转发到个人飞书给贾维斯
3. **表格分享**：如果公司允许跨账号分享文档，把工作计划表分享给个人飞书账号

**通信原理**：Mac Mini 在家里通过 WebSocket **出站连接**飞书云端（不需要公网IP，不需要公司开放端口），你在任何设备登录个人飞书发消息，Mac Mini都能收到并回复。跟你在公司用飞书跟同事聊天是一样的网络原理。

### Mac Mini 必做设置

```bash
# 1. 设置永不休眠（仅关闭显示器）
sudo pmset -a sleep 0 disksleep 0 displaysleep 10 tcpkeepalive 1 ttyskeepawake 1 autopoweroff 0 powernap 0

# 2. 安装 Homebrew（国内镜像加速）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
echo 'export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.aliyun.com/homebrew/homebrew-bottles' >> ~/.zshrc
source ~/.zshrc

# 3. 安装核心依赖
brew install node@22 python@3.9 git ffmpeg imagemagick
echo 'export PATH="/usr/local/opt/node@22/bin:$PATH"' >> ~/.zshrc
echo 'export PATH="/usr/local/opt/python@3.9/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

### 生图三层策略（零额外成本为主）

```
第一层（主力/免费）：Codex Computer Use 操控 ChatGPT 网页版
  → 夜间流水线自动生成封面+配图，用Plus订阅权益，不花API钱
  → 每张间隔30-60秒随机延迟，避免触发风控
  → 自动下载到NAS素材目录

第二层（人工精修）：你早上审核时
  → 不满意的图，打开ChatGPT App/网页手动调整prompt
  → Gemini做备选风格对比
  → 确认后替换终稿

第三层（API兜底）：TinyRouter接入GPT-Image-2 API
  → Low档$0.006/张(约4分钱)，Medium档$0.053/张
  → 网页版自动化故障时自动fallback
  → 批量测试时用API更稳定
```

### 工作任务看板模块

基于飞书多维表格 + 定时Cron + 本地HTML仪表盘，实现工作任务自动聚合与可视化：

| 组件 | 实现方式 | 说明 |
|------|----------|------|
| 数据源 | 飞书多维表格API（读取工作计划表、任务表、项目表） | 你现有飞书里的工作计划表格无需改动 |
| 数据采集 | OpenClaw/Hermes Cron每30分钟读取一次 | 自动识别本月/本周/今日任务、即将到期节点、待提交资源 |
| 智能分析 | AI自动计算优先级、识别风险、生成建议 | 哪些任务快到期、哪些项目进度滞后、时间冲突预警 |
| HTML仪表盘 | 自动生成单文件HTML页面，存在NAS | 暗色主题，响应式布局，手机/电脑均可访问 |
| 访问方式 | Mac Mini起本地HTTP服务，Tailscale内网访问 | http://macmini-ip:8080/贾维斯工作台.html |
| 飞书推送 | 每天早上8:30推送今日任务摘要+到期提醒 | 交互式卡片，紧急任务标红 |
| 提醒机制 | 到期前3天/1天/当天分级提醒 | 飞书消息+飞书Bot @提醒 |

仪表盘包含模块：紧急待办统计、今日任务清单、本周重点、待提交资源时间线、项目进度条、本月里程碑、AI智能建议。

## 四、分阶段实施计划

### Phase 1：基础环境搭建（第1周）

**目标：飞书发消息，AI能回复**

- [ ] Mac Mini 设置永不休眠，安装 Homebrew + Node.js 22 + Python 3.9 + Git
- [ ] 安装 Tailscale（Mac Mini + 公司Win + 手机），测试远程访问
- [ ] 安装 TinyRouter
  ```bash
  # 下载编译或下载release二进制
  # 浏览器打开 http://localhost:20128
  # Web UI 中添加所有 API Key（DeepSeek/OpenAI/Anthropic/中转站）
  # 配置前缀：ds=DeepSeek, oa=OpenAI, ac=Anthropic
  ```
- [ ] 安装 OpenClaw
  ```bash
  git clone https://gitee.com/openclaw/openclaw.git
  cd openclaw
  python3 -m venv venv
  source venv/bin/activate
  pip3 install -r requirements.txt --index-url=https://pypi.tuna.tsinghua.edu.cn/simple
  npm install --registry=https://registry.npmmirror.com
  npm run onboard
  # 配置 baseUrl 指向 http://localhost:20128/v1
  ```
- [ ] 飞书开放平台创建应用（使用**个人飞书账号**，不是公司企业账号）
  - 注册个人飞书账号（如果没有的话）
  - 在 open.feishu.cn 创建机器人应用
  - 开启机器人能力、消息接收、云文档/多维表格权限
- [ ] OpenClaw 接入飞书通道，测试发消息回复
- [ ] 配置 OpenClaw 开机自启

**验收标准：在飞书上给机器人发"你好"，能收到AI回复**

### Phase 2：灵感管理系统（第2周）

**目标：飞书发灵感，AI自动分类存储到多维表格**

- [ ] 创建个人飞书多维表格（灵感库+工作任务表）
  - 灵感库字段：灵感内容、类型（想法/选题/金句/待办/笔记）、分类标签、状态、AI扩写、入库时间
  - 工作任务字段：任务名、截止日期、优先级、项目名、状态、来源、提醒设置
- [ ] 配置 AI 工作流
  - 灵感分类工作流：收到消息时自动识别灵感类型→写入灵感库→回复确认
  - 任务录入工作流：收到"XX时间交XX"类消息时自动识别为任务→写入任务表→设置提醒
- [ ] 安装 Hermes Agent
  ```bash
  curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
  hermes setup
  # 配置模型通过 TinyRouter 访问
  hermes gateway setup  # 暂不配置飞书通道，通过OpenClaw调度
  ```
- [ ] 配置 OpenClaw 按需调度 Hermes（通过 MCP 或 Skill）
- [ ] 搭建 LLM Wiki 目录结构（参考 Karpathy 方案）
  ```
  ~/wiki/
  ├── raw/          # 原始素材（只读）
  ├── wiki/         # AI维护的知识页
  ├── index.md      # 内容索引
  └── log.md        # 操作日志
  ```

**验收标准：飞书发一个灵感，自动分类存入多维表格，可在飞书中查看**

### Phase 2.5：工作任务看板（第2-3周，与Phase 3并行）

**目标：从飞书工作计划表格自动聚合任务，生成可视化仪表盘，一目了然把握工作全貌**

- [ ] 梳理个人飞书中的工作任务表格结构
  - 数据来源：你通过飞书消息快捷录入的工作任务（"周五交Q3方案"等）
  - 可选：如果公司飞书允许跨账号分享，可读取分享的工作计划表
  - 字段：任务名、截止日期、负责人、项目名、状态、优先级
  - 配置飞书API读取权限（个人飞书开放平台已开通）
- [ ] 创建 work-dashboard Skill
  - 读取飞书多维表格任务数据（每30分钟Cron执行一次）
  - 自动分类：今日/本周/本月/即将到期/已逾期
  - AI分析：优先级排序、进度计算、风险预警、时间冲突检测
  - 生成智能建议（先做什么、什么快到期、什么有风险）
- [ ] 自动生成HTML仪表盘页面
  - 暗色主题，响应式设计（手机/电脑/平板均可）
  - 模块：统计卡片（紧急/今日/本周/里程碑/待提交）、今日任务清单、本周重点、到期时间线、项目进度条、本月里程碑、AI建议
  - 实时时钟显示，每30分钟自动刷新数据
  - 单HTML文件，保存到NAS共享目录
- [ ] Mac Mini 启动本地HTTP服务
  ```bash
  cd /Volumes/NAS/dashboard && python3 -m http.server 8080
  # 通过Tailscale访问：http://macmini-tailscale-ip:8080/贾维斯工作台.html
  ```
- [ ] 配置飞书每日工作提醒（Cron每天8:30）
  - 推送今日任务摘要卡片（紧急任务标红）
  - 列出今天到期/明天到期的提交项
  - 附仪表盘链接，手机可直接打开
- [ ] 配置分级到期提醒
  - 截止前3天：黄色提醒"还有3天到期"
  - 截止前1天：橙色提醒"明天截止"
  - 截止当天：红色@提醒"今天截止"
  - 已逾期：红色警告标记

**验收标准：打开仪表盘页面能看到实时任务统计，每天早上飞书收到今日任务推送，到期任务自动提醒**

### Phase 4：主动选题引擎（第4周）

**目标：每天早上8点，AI主动推送3个选题供你选择**

- [ ] 安装卡兹克开源的 aihot Skill（AI领域热点，免费无需API Key）
  ```bash
  curl -fsSL https://aihot.virxact.com/aihot-skill/install.sh | bash
  # 触发方式："今天AI圈有什么新东西"、"最近一周的AI论文"
  ```
- [ ] 配置 Cron 定时任务
  ```
  # Hermes 自然语言配置：
  "每天早上8点，运行选题引擎，分析热点+我的历史数据，
   生成3个选题推送到飞书，带交互式选择按钮"
  ```
- [ ] 接入五层信源架构
  - 第一层（全社会热点）：微博/知乎/抖音/百度热榜API（tianapi/xxapi，免费）
  - 第二层（垂直领域）：aihot Skill（AI领域）+ RSSHub订阅竞品公众号/小红书
  - 第三层（内部数据）：历史文章数据（飞书多维表格）+ LLM Wiki灵感库 + 评论区挖掘
  - 第四层（时间季节）：节假日日历 + 历史同期表现 + Google Trends趋势
  - 第五层（长尾探索，可选）：Twitter KOL / Hacker News / arXiv论文
- [ ] 实现选题评分算法
  ```
  潜力分 = 热度增速×0.4 + 垂直匹配×0.3 + 低竞争度×0.2 + 互动潜力×0.1
  7:2:1原则：70%干货实用 + 20%痛点情绪 + 10%纯热点
  去重：近7天发过类似角度的直接过滤
  ```
- [ ] 飞书交互式消息卡片（带「选这个」「换一批」「今天休息」按钮）
- [ ] 配置反馈闭环：发布后回收数据，更新评分权重

**验收标准：每天早上8点飞书自动收到3个选题卡片，点按钮即可触发写作**

### Phase 4.5：即梦动画提示词生成（第4-5周，与Phase 5并行）

**目标：飞书发动画大纲/灵感，AI自动生成即梦Seedance 2.0可用的标准化提示词**

- [ ] 创建 jimeng-prompt Skill，内置即梦八层框架
  - 第1层：素材角色声明（@Image/@Video/@Audio锁定素材）
  - 第2层：镜头标签（镜头1/镜头2...，2-4个镜头/段，5分钟动画约20-30段）
  - 第3层：景别与主体（全景/中景/特写 + 2-3个具体静态特征）
  - 第4层：动作（身体部位级描述 + 情绪外化，避免"快速/fast"）
  - 第5层：运镜（每镜头1个运镜，中文+英文术语，如"缓慢推进dolly in"）
  - 第6层：场景与光影（环境 + 光源方向/色温 + 氛围元素）
  - 第7层：音频（对话/音效/环境音，镜头间音频渐弱渐入）
  - 第8层：全局收尾（风格锚点 + 约束词模板 + 4K/16:9/时长）
- [ ] 内置约束词固定模板
  ```
  保持面部一致，无变形，无拉伸，避免身份漂移，避免抖动和弯曲肢体，
  不生成字幕，不生成水印，不生成Logo
  ```
- [ ] 内置运镜词汇表（dolly in/pan/tracking/orbit/handheld/fixed等）
- [ ] 内置情绪外化映射表（悲伤→低下头肩膀发抖；喜悦→嘴角上扬脚步轻快等）
- [ ] 飞书交互：生成提示词后提供[复制到即梦] [调整风格] [生成角色参考图]按钮
- [ ] （进阶）Codex Computer Use自动打开即梦网页粘贴提示词

**验收标准：飞书发一段动画故事大纲，自动输出符合即梦八层框架的分镜提示词，复制到即梦可直接生成**

## 五、日常使用流程

### 场景1：睡前投灵感

```
23:00 你在飞书发："明天写一篇关于AI Agent普通人怎么用的文章"
  → OpenClaw 接收，识别为选题
  → 存入多维表格选题池
  → 回复："选题已记录，明早8点给你三版初稿"
```

### 场景2：早上选题推送

```
08:00 飞书自动收到消息卡片：
  "早安！今天推荐3个选题：
   ① AI Agent普通人怎么用？（匹配度92%）
   ② ...
   ③ ...
   [选①] [选②] [选③] [换一批] [自己写]"

你点击 [选①]
  → 自动触发三模型并行写作
  → 30-60分钟后飞书通知："三版稿件+配图已就绪"
```

### 场景3：审核修改

```
09:00 你打开飞书云文档
  → 对比三版文章
  → 选择一版或融合修改
  → 手动润色，形成终稿
  → 发消息"可以发布了"
```

### 场景4：一键发布

```
你发"发布"
  → wenyan-md 自动排版
  → 推送公众号草稿箱
  → 视频脚本生成→渲染MP4→推送视频号草稿
  → 小红书适配版生成→推送草稿
  → 飞书通知："已推送到各平台草稿箱，请审核后发布"
```

### 场景5：出差/忙碌

```
你发"我出差3天"
  → 进入缺席模式
  → AI自动选最优选题执行
  → 写完存草稿，飞书推送链接
  → 你手机快速浏览，点"批准"或"改改"
  → 不会自动发布，安全兜底
```

### 场景6：动画灵感转即梦提示词

```
你在飞书发："做一个5分钟小动画，讲上班族发现AI是去世妻子留下的程序"
  → OpenClaw识别为动画需求
  → 调用jimeng-prompt Skill
  → 拆解为20-30个镜头，每个镜头按八层框架生成提示词
  → 飞书推送分镜表+完整提示词，附[复制到即梦][调整风格]按钮
  → 你复制到即梦网页，或让Codex自动粘贴
```

### 场景7：每日工作看板

```
08:30 飞书自动推送今日工作摘要卡片：
  "早上好！今天有5项任务，其中2项紧急：
   🔴 项目A设计稿（今天18:00截止）
   🟡 周报提交（明天10:00截止）
   [打开工作台] [查看详情]"
09:00 你打开工作台HTML页面（电脑或手机）
  → 一目了然看到所有任务、项目进度、即将到期事项
  → AI建议区域告诉你优先做什么、有什么风险
全天 飞书实时提醒
  → 任务截止前3天/1天/当天分级提醒
  → 你完成任务在飞书表格里标记，30分钟内仪表盘自动更新
```

## 七、避坑指南

### 安全相关
- [ ] **不要把 OpenClaw/Hermes 管理端口直接暴露公网**（国家互联网应急中心已发安全警告）
- [ ] 使用 Tailscale 访问仪表盘，不要做端口映射
- [ ] TinyRouter 本地运行，不对外暴露
- [ ] API Key 只存 TinyRouter，各组件统一从本地调用
- [ ] 给 OpenClaw 配置 Token 认证
- [ ] **使用个人飞书账号创建机器人**，不要用公司企业飞书（避免公司审计风险，无需IT审批）
- [ ] 公司电脑用飞书网页版登录个人账号与贾维斯对话，不影响公司飞书正常使用
- [ ] 工作任务通过快捷消息录入（"周五交XX"），不自动抓取公司飞书数据

### 公众号发布
- [ ] 公众号 API 要求 IP 白名单，初期手动复制发布即可
- [ ] 后期固定代理节点或用云服务器中转
- [ ] 永远推送到草稿箱，不要自动直接发布
- [ ] 封面图尺寸：900×383px（大图2.35:1），小图383×383px

### 双Agent共存
- [ ] OpenClaw 端口：18789
- [ ] Hermes 端口：26002
- [ ] TinyRouter：20128
- [ ] 配置目录天然隔离：`~/.openclaw/` 和 `~/.hermes/`
- [ ] 只让 OpenClaw 接飞书，Hermes 通过 OpenClaw 调度

### 成本控制
- [ ] 日常任务用 DeepSeek（便宜）
- [ ] 专业写作/编程才调用 Claude/Codex
- [ ] 生图主力用Codex操控ChatGPT网页版（Plus订阅已包含，不额外花钱）
- [ ] 生图API仅做兜底备用（Low档4分钱一张，月成本极低）
- [ ] Gemini免费额度作为风格备选
- [ ] TinyRouter 配置 fallback：先试便宜模型，失败再用好模型
- [ ] 设置 Token 消耗上限，防止意外烧钱

### 生图注意事项
- [ ] ChatGPT网页版自动化不要太频繁，每张间隔30-60秒随机延迟
- [ ] 不要一秒钟生成多张，避免触发账号风控
- [ ] 图片生成后用ImageMagick二次裁剪到标准尺寸，不依赖AI输出精确尺寸
- [ ] 需要在图上渲染文字优先用GPT-Image-2（文字渲染能力最强）
- [ ] 纯概念图/风格化图可以尝试Gemini做备选

### 稳定性
- [ ] Mac Mini 配置永不休眠
- [ ] NAS 定期备份配置文件和 Wiki 数据
- [ ] 绿联系统更新前备份 OpenClaw 配置
- [ ] 重要数据（文章/素材/Wiki）存在 NAS 而非 Mac 本地
