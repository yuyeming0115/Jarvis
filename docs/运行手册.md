# 运行手册

## 启动工作台

```bash
bash "$HOME/Jarvis/services/start-workbench.sh"
```

默认地址：

```text
http://127.0.0.1:8080/
```

如 8080 被占用，可以临时换端口：

```bash
JARVIS_WORKBENCH_PORT=8081 bash "$HOME/Jarvis/services/start-workbench.sh"
```

## 检查健康状态

```bash
bash "$HOME/Jarvis/services/health-check.sh"
```

## 停止工作台

```bash
bash "$HOME/Jarvis/services/stop-workbench.sh"
```

停止脚本只会停止 Jarvis 启动脚本记录的工作台进程。如果端口被其他程序占用，脚本会提示，不会自动结束未知进程。

如果工作台已经交给 launchd 管理，使用下面的命令卸载常驻服务：

```bash
bash "$HOME/Jarvis/services/uninstall-launchd.sh"
```

## 备份 JSON

```bash
bash "$HOME/Jarvis/services/backup-json.sh"
```

备份会写入：

```text
$HOME/Jarvis/backups/
```

## 初始化 SQLite

```bash
bash "$HOME/Jarvis/services/init-sqlite.sh"
```

首次运行会从当前 `apps/workbench/data/*.json` 初始化 SQLite。之后 SQLite 是主数据源。

## 导出 JSON

```bash
bash "$HOME/Jarvis/services/export-json.sh"
```

导出会从 SQLite 读取数据，写入 `backups/export-json-*`。

## 备份 SQLite

```bash
bash "$HOME/Jarvis/services/backup-db.sh"
```

备份会复制 `backend/db/jarvis.sqlite3` 以及可能存在的 WAL 文件。

## V1.1 API

工作台启动后，本地 API 和页面共用一个端口：

```text
http://127.0.0.1:8080/
```

主要接口：

```text
GET    /api/tasks
POST   /api/tasks
PATCH  /api/tasks/{task_id}
POST   /api/tasks/{task_id}/complete
GET    /api/ideas
POST   /api/ideas
GET    /api/topics
POST   /api/topics
GET    /api/messages
GET    /api/system-status
GET    /api/logs
POST   /api/inbox
POST   /api/feishu/event
```

每次写入 JSON 前，后端会自动在 `backups/` 下创建快照。

V1.2 开始，写入前会自动备份 SQLite，并导出一份 JSON 快照。

## Launchd 服务

安装开机自启和每日维护：

```bash
bash "$HOME/Jarvis/services/install-launchd.sh"
```

查看状态：

```bash
bash "$HOME/Jarvis/services/status-launchd.sh"
```

卸载：

```bash
bash "$HOME/Jarvis/services/uninstall-launchd.sh"
```

安装后会创建两个用户级 LaunchAgent：

```text
~/Library/LaunchAgents/com.local.jarvis.workbench.plist
~/Library/LaunchAgents/com.local.jarvis.maintenance.plist
~/Library/LaunchAgents/com.local.jarvis.reminders.plist
~/Library/LaunchAgents/com.local.jarvis.feishu-public-monitor.plist
```

工作台和提醒服务会开机自启并崩溃自动重启。飞书公网监控会每 5 分钟检查一次临时回调链路。维护任务每天 03:15 执行，内容包括 SQLite 备份、JSON 导出和日志轮转。

## 本地提醒

提醒服务会扫描未完成任务的 `due_at`，到期后通过 macOS 本地通知提醒一次。

查看提醒服务状态：

```bash
bash "$HOME/Jarvis/services/status-launchd.sh"
```

手动干跑一次提醒扫描，不弹通知：

```bash
cd "$HOME/Jarvis"
python3 -m backend.core.reminders --once --dry-run
```

测试时间解析：

```bash
cd "$HOME/Jarvis"
python3 -m backend.core.reminders --parse "明天 10 点提醒我复盘 Jarvis"
```

## 日志轮转

手动执行：

```bash
bash "$HOME/Jarvis/services/rotate-logs.sh"
```

默认单个日志超过 1MB 才轮转，最多保留 10 个轮转日志。

## 飞书本地模拟

```bash
bash "$HOME/Jarvis/services/simulate-feishu-event.sh" challenge
bash "$HOME/Jarvis/services/simulate-feishu-event.sh" task
bash "$HOME/Jarvis/services/simulate-feishu-event.sh" idea
bash "$HOME/Jarvis/services/simulate-feishu-event.sh" topic
```

模拟消息会写入 `messages` 表，并按规则进入任务、灵感或选题。

飞书完整回归检查：

```bash
bash "$HOME/Jarvis/services/verify-feishu-flow.sh"
```

该脚本会模拟一条飞书提醒消息，检查任务和 `due_at` 是否生成，然后自动完成测试任务。

## API 测试

只读检查：

```bash
bash "$HOME/Jarvis/services/api-smoke-test.sh"
```

写入检查：

```bash
bash "$HOME/Jarvis/services/api-smoke-test.sh" --write
```

写入检查会创建并完成一条测试任务，同时触发自动备份。

## V4.0 完整 smoke test

前端与 API 基础验证：

```bash
python -m py_compile backend/main.py backend/core/topics.py backend/core/drafts.py
node --check apps/workbench/app.js
```

完整内容生产链路验证：

```bash
bash "$HOME/Jarvis/services/api-smoke-test.sh" --write
bash "$HOME/Jarvis/services/drafts-api-smoke-test.sh"
bash "$HOME/Jarvis/services/content-generation-smoke-test.sh"
bash "$HOME/Jarvis/services/content-review-smoke-test.sh"
bash "$HOME/Jarvis/services/wiki-archive-smoke-test.sh"
bash "$HOME/Jarvis/services/content-smoke-test.sh"
bash "$HOME/Jarvis/services/content-regression-test.sh"
```

在 Windows 本地验证时，优先确认 `8080` 端口只存在一个 Jarvis 后端监听进程，并且 `logs/workbench.pid` 与实际监听 PID 一致。若 Bash 脚本受 WSL、CRLF 或 `python` 命令差异影响，可使用 Windows 原生 HTTP 调用覆盖同等 API 链路；验证范围应至少包含：

- 前端 `/`、`/app.js`、`/styles.css` 可访问。
- `api-smoke-test --write` 等价写入检查。
- 草稿 CRUD。
- 选题生成草稿。
- 草稿提交审核、批准和删除。
- 未批准草稿禁止归档，批准草稿可归档到 Wiki。
- 完整链路：选题 -> 草稿 -> 编辑 -> 审核 -> 批准 -> 归档。
- 回归检查：旧 API、内容 API、无公网发布接口。
