# 运行手册

## 启动工作台

```bash
bash "$HOME/Jarvis/services/start-workbench.sh"
```

默认地址：

```text
http://127.0.0.1:8080/
```

如 8080 被占用，可以临时换端口：

```bash
JARVIS_WORKBENCH_PORT=8081 bash "$HOME/Jarvis/services/start-workbench.sh"
```

## 检查健康状态

```bash
bash "$HOME/Jarvis/services/health-check.sh"
```

## 停止工作台

```bash
bash "$HOME/Jarvis/services/stop-workbench.sh"
```

停止脚本只会停止 Jarvis 启动脚本记录的工作台进程。如果端口被其他程序占用，脚本会提示，不会自动结束未知进程。

## 备份 JSON

```bash
bash "$HOME/Jarvis/services/backup-json.sh"
```

备份会写入：

```text
$HOME/Jarvis/backups/
```

## V1.1 API

工作台启动后，本地 API 和页面共用一个端口：

```text
http://127.0.0.1:8080/
```

主要接口：

```text
GET    /api/tasks
POST   /api/tasks
PATCH  /api/tasks/{task_id}
POST   /api/tasks/{task_id}/complete
GET    /api/ideas
POST   /api/ideas
GET    /api/topics
POST   /api/topics
GET    /api/system-status
GET    /api/logs
POST   /api/inbox
```

每次写入 JSON 前，后端会自动在 `backups/` 下创建快照。

## API 测试

只读检查：

```bash
bash "$HOME/Jarvis/services/api-smoke-test.sh"
```

写入检查：

```bash
bash "$HOME/Jarvis/services/api-smoke-test.sh" --write
```

写入检查会创建并完成一条测试任务，同时触发自动备份。
