# 飞书接入清单

## 当前 V2.0 已完成

- 本地事件回调接口：`POST /api/feishu/event`
- 本地 URL 验证 challenge
- 明文文本消息解析
- 规则分类入库
- 消息记录表 `messages`
- 本地模拟脚本
- 临时 HTTPS 隧道测试脚本
- 飞书官方 CLI：`lark-cli`

## 你需要手动准备

1. 登录飞书开放平台。
2. 创建自建应用或机器人应用。
3. 获取 `APP_ID`、`APP_SECRET`、`Verification Token`。
4. 先不要配置 `Encrypt Key`，V2.0 本地适配器暂不解密加密事件。
5. 准备一个安全的 HTTPS 回调地址。

## 本地配置

真实密钥只放在本机 `.env`，不要粘贴到聊天窗口或 Git：

```bash
cp "$HOME/Jarvis/adapters/feishu/feishu.env.example" "$HOME/Jarvis/adapters/feishu/feishu.env"
```

Jarvis 服务会读取：

```text
$HOME/Jarvis/.env
```

建议把飞书开放平台里的 Verification Token 写到 `.env`：

```bash
FEISHU_VERIFICATION_TOKEN=你的飞书 Verification Token
```

修改 `.env` 后重启工作台服务：

```bash
launchctl kickstart -k "gui/$UID/com.local.jarvis.workbench"
```

## 安全边界

- 不暴露 Mac Mini 管理端口。
- 不直接把 `127.0.0.1:8080` 暴露公网。
- 不抓取公司飞书或公司系统数据。
- 只处理你发给 Jarvis 机器人的消息。
- 外部回调方案需要单独确认后再启用。

## 临时公网回调测试

启动只转发飞书事件的本地 relay：

```bash
bash "$HOME/Jarvis/services/start-feishu-relay.sh"
```

启动临时 HTTPS 隧道：

```bash
bash "$HOME/Jarvis/services/start-feishu-tunnel.sh"
```

查看飞书开放平台应填写的完整请求地址：

```bash
bash "$HOME/Jarvis/services/show-feishu-callback-url.sh"
```

停止临时隧道：

```bash
bash "$HOME/Jarvis/services/stop-feishu-tunnel.sh"
bash "$HOME/Jarvis/services/stop-feishu-relay.sh"
```

注意：临时隧道 URL 每次重启可能变化，只适合测试；长期方案应使用固定域名或云端轻量中转。

当前机器已安装飞书官方 CLI：

```bash
lark-cli --version
```

## 飞书开放平台最小配置

1. 打开飞书开放平台，创建自建应用。
2. 进入事件订阅。
3. 请求地址填写 `show-feishu-callback-url.sh` 输出的 HTTPS 地址。
4. 暂时不要填写 Encrypt Key。
5. 事件选择 `im.message.receive_v1`。
6. 给应用机器人发送消息，回到 Jarvis 工作台查看任务、灵感或选题。
