# OpenClaw 与 Hermes 规划

## 当前结论

暂不在 V2.0 安装 OpenClaw 和 Hermes。

原因：

- V2.0 的关键目标是验证飞书入口能稳定入库。
- OpenClaw 会引入额外工作流调度和模型调用变量。
- Hermes 是长期记忆和知识库能力，依赖稳定的数据结构和内容沉淀。

## 推荐安装时机

### OpenClaw

建议在 V2.1 提醒闭环稳定后评估安装。

安装前应满足：

- 飞书消息能稳定进入 `messages`。
- 任务、灵感、选题能稳定入库。
- 每日摘要和到期提醒规则已明确。
- 失败日志和重试机制能查。

### Hermes

建议在 V3.2 知识库版开始安装测试。

安装前应满足：

- Wiki 目录开始积累真实内容。
- 选题和草稿有稳定归档格式。
- 用户写作偏好和长期记忆的更新规则已明确。

## 当前替代方案

V2.0 到 V2.1 先用 Jarvis 自带的本地 Gateway 和规则分类，不引入外部 Agent 框架。
