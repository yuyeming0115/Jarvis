# 变更记录

## 2026-07-04

### V2.1 飞书入口与提醒闭环版

- 跑通个人飞书自建应用真实消息回调。
- 新增提醒时间解析，支持 `今天`、`明天`、`后天`、`周五`、`10 点`、`9:30` 等常见表达。
- 飞书和 Inbox 入口创建任务时会写入 `due_at` 和提醒级别。
- 新增 `reminder_notifications` 表，避免同一任务重复提醒。
- 新增 macOS 本地提醒服务，并通过 launchd 常驻运行。
- 新增飞书公网回调监控，每 5 分钟检查 relay/tunnel 是否可用。
- 飞书临时 tunnel 会优先复用已有 `loca.lt` 子域名，降低回调地址变化概率。
- 新增飞书链路回归脚本，验证消息、任务和提醒时间生成后自动完成测试任务。
- 回填已有未完成飞书提醒任务的 `due_at`。

## 2026-07-03

### V2.0 飞书入口准备版

- 新增 `POST /api/feishu/event` 本地飞书事件回调接口。
- 支持飞书 URL 验证 challenge。
- 支持明文文本消息事件解析和 Verification Token 校验。
- 暂不处理 Encrypt Key 加密事件，真实接入前需保持明文回调或后续补加解密。
- 新增 `messages` 表记录外部入口原始消息和分类结果。
- 飞书消息按规则分类为任务、灵感或选题并入库。
- 新增飞书本地模拟脚本和接入清单。
- 新增只转发飞书事件的本地 relay 和临时 HTTPS 隧道脚本。
- 安装飞书官方 CLI：`lark-cli`。
- 明确 OpenClaw 和 Hermes 后续接入时机。

### V1.3 本地服务化版

- 新增工作台 launchd 常驻服务配置。
- 新增每日维护 launchd 配置，每天 03:15 执行备份和日志轮转。
- 新增 `run-workbench.sh`，供 launchd 以前台进程方式运行后端。
- 新增 `install-launchd.sh`、`uninstall-launchd.sh`、`status-launchd.sh`。
- 新增 `run-maintenance.sh` 和 `rotate-logs.sh`。
- `backup-db.sh` 改用 SQLite backup API 生成一致性备份。
- 安装 launchd 后，`stop-workbench.sh` 会提示使用卸载脚本，避免 KeepAlive 立即重启造成误解。

### V1.2 SQLite 数据库版

- 新增 SQLite schema 和 `backend/db/jarvis.sqlite3` 主数据源。
- 后端数据层从 JSON 读写升级为 SQLite 读写。
- 首次初始化会从现有 JSON 数据迁移到 SQLite。
- 新增 `init-sqlite.sh`、`export-json.sh`、`backup-db.sh`。
- `backup-json.sh` 改为从 SQLite 导出 JSON。
- 写入前自动备份 SQLite，并导出 JSON 快照。

### V1.1 本地可编辑版

- 新增 Python 标准库后端 API，不安装外部依赖。
- 工作台改为通过 `/api` 读取和写入 JSON。
- 支持新增任务、灵感、选题。
- 支持修改任务状态、优先级和标记完成。
- 后端写入前自动创建 JSON 快照备份。
- 新增 API 冒烟测试脚本，支持只读检查和可选写入检查。
- 新增备份与 Git 策略文档。
- 新增 `.gitignore`，排除真实密钥、本地 JSON 数据、日志、PID、备份和数据库文件。

### V1.0 本地展示版

- 创建 Jarvis V1.0 本地展示版目录结构。
- 创建工作台静态页面、样式和数据读取脚本。
- 创建本地 JSON 示例数据：任务、灵感、选题、系统状态、运行日志。
- 创建启动、停止、健康检查和 JSON 备份脚本。
- 启动脚本会等待本地 HTTP 服务可用；停止脚本只停止已记录的 Jarvis 工作台进程。
- 创建 `.env.example` 和 launchd 示例文件。
- 创建运行手册、数据结构、多入口接入策略和安全边界文档。
