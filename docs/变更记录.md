### V2.4 微信评估版 + V3.0 AI 分类版

  - 更新了 `prompts/classify-message.md` 分类提示词

  - AI 分类会自动提取更简洁的标题（不再用整段原文作为标题）

### V2.3 Telegram 备用入口版

- 新增 Telegram Bot 适配器，作为飞书之外的备用远程入口。
- Bot 采用长轮询（getUpdates）方式工作，不需要公网回调地址。
- 支持命令：
  - `/start`、`/help` - 显示帮助
  - `/add <内容>` - 新增任务（支持自然语言时间解析）
  - `/idea <内容>` - 记录灵感
  - `/topic <内容>` - 记录选题
  - `/today` - 查看今日待办列表
  - `/status` - 查看系统状态和统计
- 支持 `TELEGRAM_ALLOWED_USER_IDS` 白名单，只有指定用户才能使用。
- 未配置 `TELEGRAM_BOT_TOKEN` 时自动跳过不启动，不影响主服务。
- 后端启动/停止时自动管理 Bot 线程生命周期。
- 所有消息统一走 Inbox 路由，和飞书数据互通。

### V2.2 飞书按钮版 + Windows 兼容

- 新增跨平台路径自动检测，不再硬编码 `~/Jarvis`，支持 `JARVIS_ROOT` 环境变量自定义位置。
- 新增 Windows 兼容：PowerShell 启动/停止/健康检查脚本（`start-workbench.ps1`、`stop-workbench.ps1`、`health-check.ps1`）。
- 新增跨平台通知支持：macOS 继续用 osascript，Windows 自动降级（未来可接入 winrt Toast）。
- 新增飞书交互式卡片支持：任务/灵感/选题入库后自动发送带按钮的确认卡片。
- 新增飞书卡片按钮回调处理，支持动作：
  - ✅ 完成任务
  - ⏰ 稍后提醒（1小时/明天）
  - 📋 查看待办列表
  - 📰 灵感转选题
  - 📋 灵感转任务/选题转任务
- 新增飞书 API 客户端模块（`feishu_client.py`），支持获取 tenant_access_token 和发送消息卡片。
- 新增卡片模板模块（`feishu_cards.py`）和卡片动作处理模块（`feishu_card_actions.py`）。
- 新增 `topics.py` 的 `patch_topic` 函数。
- 统一 `/api/feishu/event` 入口同时处理事件回调和卡片按钮回调。

### V2.1 飞书入口与提醒闭环版

- 跑通个人飞书自建应用真实消息回调。
