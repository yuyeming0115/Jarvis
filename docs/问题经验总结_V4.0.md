# 问题经验总结 V4.0 — 内容生产 MVP 迭代

> 记录 V4.0 迭代过程中遇到的问题、排查思路和最终解决方案，供后续迭代参考。

---

## 1. 图片生成 HTTP 405 Method Not Allowed

**现象**
```
POST http://localhost:20129/v1/images/generations
→ HTTP 405 Method Not Allowed
```

**排查**
- TinyRouter 是 LLM 文本 API 网关，只代理 `/chat/completions` 类接口
- `/images/generations` 是 OpenAI 图片生成接口，TinyRouter **未注册此路由**
- Jarvis 错误地将图片生成也走 TinyRouter 统一入口

**原因**
`llm.py` 的 `generate_image()` 使用了 `_get_llm_base_url()`（即 TinyRouter 地址），而不是独立的图片 API 地址。

**解决**
- `generate_image()` 改为使用 `_get_image_gen_base_url()`（独立配置项）
- 恢复设置面板中的「图片生成配置」分组（`image` 分组之前被误设为 `null`）

**经验**
> 统一 API 入口 ≠ 所有功能都走同一个 URL。文本和图片是两套不同的 API，要分开配置。

## 2. Agnes AI 内容安全拦截（content_policy_violation）

**现象**
```
HTTP 400 · content_policy_violation
"Unable to generate this content. Please modify your prompt and try again."
```

**排查**
- 提示词含"数字人""毛笔""水墨山水"等词，触发 Agnes 服务端内容审核策略
- 这是 **Agnes 服务端的限制**，不是 Jarvis 代码问题

**解决**
实现三级容错机制（`llm.py` 的 `generate_image()`）：
1. **第 1 次**：原始提示词
2. **第 2 次**：自动清洗敏感词（`_clean_image_prompt()`）
   - 移除：数字人、真人、肖像、政治、暴力、色情、赌博、毒品等关键词
   - 移除：8K 超高清等技术参数（容易触发误判）
3. **第 3 次**：激进简化（`_simplify_prompt()`）
   - 截断到核心描述（前 200 字）
   - 去掉负面提示词
   - 使用更通用的模型重试

**经验**
> 第三方 API 要有容错机制。自动化流水线运行时遇到内容审核拦截，不能静默卡住，要自动修正并继续执行。

## 3. 中文请求体编码问题（Windows）

**现象**
```bash
curl -X POST http://127.0.0.1:8080/api/topics \
  -H "Content-Type: application/json" \
  -d '{"title":"测试选题",...}'
→ HTTP 500 · 'utf-8' codec can't decode byte 0xb2
```

**排查**
- Windows Git Bash 的 curl 发送中文时，请求体编码不是 UTF-8
- `read_payload()` 直接用 `decode("utf-8")` 会抛异常

**解决**
修复 `main.py` 的 `read_payload()`：
```python
try:
    raw = self.rfile.read(length)
    payload = json.loads(raw.decode("utf-8"))
except UnicodeDecodeError:
    payload = json.loads(raw.decode("gbk", errors="replace"))
except json.JSONDecodeError as error:
    raise ApiError(f"JSON 解析失败：{error}") from error
```

**经验**
> Windows 环境要做更多的编码容错。生产环境建议统一用 Python `requests` 或 `urllib` 发请求，避免 curl 编码问题。

## 4. AI生图不能智能调用已存储的提示词

**现象**
点击「🤖 AI文生图」按钮，提示词输入框始终为空，用户需要重新输入。

**原因**
`openImageGenModal("")` 始终传空字符串，没有读取已选中的媒体提示词。

**解决**
1. 修改 `openImageGenModal(defaultPrompt)`：
   - 若 `defaultPrompt` 为空，自动读取 `state.mediaPrompts[0].prompt`
2. 在每个媒体提示词卡片新增「🎨 用此提示词生图」按钮
3. 点击后直接调用 `openImageGenModal(p.prompt)`

**经验**
> 用户期望"智能"——即系统应该自动使用已有的提示词，而不是每次都让用户重新输入。

## 5. 草稿预览功能缺失

**现象**
生成草稿后，只能看到纯文本预览（前 800 字），没有渲染最终效果。

**原因**
- 详情区只用 `textContent` 显示纯文本
- 没有 Markdown 渲染
- 没有「预览最终效果」入口

**解决**
1. 新增 `openDraftPreview(draft)` 函数
2. 实现 `renderMarkdownToHtml(md)` 渲染器（支持 h1~h3、粗体、斜体、代码、链接、列表）
3. 在草稿详情区新增「👁️ 预览最终效果」按钮（蓝色高亮）

**经验**
> 内容生产工具必须要有"预览最终效果"的功能。用户生成内容后，第一件事就是想看看效果。

## 6. UI 不够专业（操作按钮区）

**现象**
详情区操作按钮是 7 个紫色大按钮竖着排，视觉权重过高，不符合内容产品的阅读体验。

**原因**
所有操作按钮都用 `className: "ai-btn"` 渲染成同款大按钮。

**解决**
改成 **toolbar 风格**：
- 主要操作（生成大纲/撰写正文）：小按钮，浅色背景
- 次要操作（封面/配图/分镜/AI生图）：图标+文字，紧凑排列
- 工具操作（归档/编辑/删除）：收到底部，小字号

**经验**
> UI 要符合内容消费习惯，不是后台管理风格。操作按钮的视觉权重应该低于内容本身。

## 7. Windows 控制台 GBK 编码问题（测试脚本）

**现象**
Python 测试脚本输出 Unicode 勾号（✅）时，Windows 控制台显示乱码。

**原因**
Windows 控制台默认编码是 GBK（cp936），不支持所有 Unicode 字符。

**解决**
测试脚本输出改用 ASCII 字符：
```python
PASS = "[PASS]"
FAIL = "[FAIL]"
OK = "[OK]"
```

**经验**
> Windows 环境要避免直接输出 Unicode 特殊字符。测试脚本优先用 ASCII，确保跨平台可读。

## 8. 后端多进程冲突

**现象**
多次重启后端后，`netstat` 显示 8080 端口有多个 Python 进程监听，导致请求随机失败。

**原因**
- 每次 `python -m backend.main &` 都会启动新进程
- 旧进程没有正确停止（PID 文件与实际进程不一致）

**解决**
重启后端前，先用 Python 脚本杀掉所有占用 8080 端口的进程：
```python
import subprocess, os, signal
result = subprocess.run(['netstat', '-ano'], capture_output=True, text=True)
for line in result.stdout.splitlines():
    if ':8080' in line and 'LISTENING' in line:
        pid = line.split()[-1]
        os.kill(int(pid), signal.SIGTERM)
```

**经验**
> 重启后端前要确认旧进程已停止。建议用进程管理工具（如 `supervisor`）或 PID 文件机制来避免多进程冲突。

## 9. 测试脚本要用 Python 而不是 Bash（Windows）

**现象**
Bash 版 `content-smoke-test.sh` 在 Windows 上运行失败：
- 中文编码问题
- CRLF 换行问题
- `python` 命令指向 WSL Python 而不是 Windows Python

**解决**
创建 Python 版测试脚本：
- `services/content-smoke-test.py`
- `services/content-regression-test.py`

**经验**
> Windows 环境下优先用 Python 写测试脚本。Python 的 `urllib` 跨平台一致性更好，且能正确处理 UTF-8 编码。

## 10. 设置面板配置分组被误隐藏

**现象**
设置面板里没有「图片生成配置」区域，用户无法修改图片 API 地址和 Key。

**原因**
之前统一入口改造时，把 `SETTINGS_GROUPS` 里的 `image` 分组设为 `null`：
```javascript
image: null, // 已合并到统一入口
```

**解决**
恢复 `image` 分组：
```javascript
image: {
  label: "图片生成配置",
  fields: ["image_gen_base_url", "image_gen_api_key", "image_gen_model"]
}
```

**经验**
> 统一入口 ≠ 隐藏配置项。配置可以共用同一个 API 地址，但设置面板里还是要让用户能查看和修改。

## 附录：常见问题 FAQ

### Q1: 图片生成报 HTTP 405 怎么办？
**A**: 检查设置面板里的「图片生成 API 端点」是否填的是 Agnes AI 的地址（`https://api.agnes-ai.com/v1`），而不是 TinyRouter 地址。

### Q2: 图片生成报 HTTP 400 content_policy_violation 怎么办？
**A**: 提示词触发了 Agnes 的内容审核策略。Jarvis 会自动清洗敏感词并重试，如果还是失败，建议简化提示词（去掉"数字人"等词）。

### Q3: 如何验证 TinyRouter 配置？
**A**: 看 TinyRouter 控制台的"模型组"页面，如果显示"暂无模型组"则需要创建 `text-smart-fallback` 和 `image-smart-fallback` 两个模型组。

### Q4: 设置面板在哪打开？
**A**: 点击页面右上角的 ⚙️ 齿轮按钮。

### Q5: 如何运行完整流水线测试？
**A**: 运行 `python services/content-smoke-test.py`，全部通过后输出 `Content flow verified`。
